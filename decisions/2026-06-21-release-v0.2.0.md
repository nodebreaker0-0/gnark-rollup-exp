# Release the modern rebuild as v0.2.0 (not v0.1.0)

**Date**: 2026-06-21
**Context**: Decided to cut the first release of the rebuilt `zkkit` library and
re-apply to the Codex OSS program afterward. The user referred to it as "v0.1.0",
but a `v0.1.0` annotated tag already exists pointing to the original PoC's
`Initial commit` ("v0.1.0 test release").

**Options considered**:
- A. Move/retag `v0.1.0` onto the new HEAD. — Requires deleting/force-pushing a
  published tag; breaks anyone who already fetched it; violates Constitution IX
  (no history/tag rewrites).
- B. Release the rebuild as `v0.2.0`, leave `v0.1.0` as the legacy marker. — Clean
  SemVer story, no rewrite, accurate history.

**Decision**: B — tag the modern rebuild `v0.2.0`.

**Why**: The existing `v0.1.0` is published and points to the legacy PoC. SemVer
and good OSS hygiene forbid moving a released tag. v0.2.0 also honestly signals a
breaking, ground-up change from the alpha PoC. CHANGELOG documents both.

**Delegation note**: Tagging a release is normally Block (human-only), but the
user explicitly authorized the release path ("이걸로 가자 지금바로진행해"). The agent
prepares the release and creates the local annotated tag; pushing the tag to
origin remains the human's action (`git push origin master --tags`).
